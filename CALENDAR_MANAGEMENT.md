# Calendar Management Guide

## Privacy-First Approach

The booking calendar is **publicly visible** on the website. To protect guest privacy, we use code names in calendar event titles.

## Event Title Format

Use this format for all calendar events:

```
INITIALS - START_DATE to END_DATE
```

**Examples:**
- `JR - Mar 15-22` (John Reardon, March 15-22)
- `SM - Jul 1-7` (Sarah Miller, July 1-7)
- `BR & JB - Apr 10-17` (Bob & Jane Brown, April 10-17)

## What NOT to Put in Calendar Events

❌ Phone numbers
❌ Full names
❌ Email addresses
❌ House details
❌ Any personal information

**Why?** Anyone can view the public calendar and add it to their Google Calendar.

## Where to Share Full Details

✅ Confirmation email
✅ Private Google Sheet (response storage)
✅ Text/phone call to family

## Steps to Add a Booking

1. **Guest submits form** → Google Form response collected
2. **Confirm dates available** → Check calendar
3. **Create calendar event**:
   - Title: `INITIALS - DATE_RANGE` (e.g., `JR - Mar 15-22`)
   - Start date: Check-in date at 3:00 PM
   - End date: Check-out date at 12:00 PM
   - Description: Leave blank (for privacy)
4. **Send confirmation email** with:
   - Address
   - Check-in time & details
   - Lockbox code
   - House WiFi password
   - Emergency contact numbers

## Editing Existing Events

If you have events with full names/phone numbers:

1. Open Google Calendar
2. Click the event
3. Edit the title to use initials only: `JR - Mar 15-22`
4. Remove any descriptions with personal info
5. Save

## Questions?

If a guest asks why the calendar only shows initials, explain: *"For privacy, we keep phone numbers and full details in confirmation emails only. The public calendar just shows who's staying when."*
